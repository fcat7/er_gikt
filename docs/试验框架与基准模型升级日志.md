# 实验框架与基准模型集成记录 (Framework & Baselines Integration)

> **日期**: 2026-02-26
> **影响范围**: [🔵 ENHANCEMENT] (架构解耦、工业级调参、基准对比)

继 V2 数据流水线升级后，本项目完成了一次重大的**工程化重构**。核心目标是建立一个符合学术论文发表标准、可支持多模型公平对比、且具备自动化生产能力的实验框架。

---

## 1. 核心变更概览 (Key Milestones)

### 1.1 引入工厂设计模式 (Model Factory)
- **解耦核心训练逻辑**：通过 `ModelFactory` 统一管理模型实例化。
- **动态适配**：训练脚本不再关心模型内部构造，只需根据 `metadata.json` 自动传入参数。
- **基准模型包化**：新建 `baselines/` 包，统一收纳 `DKT`, `DKVMN`, `AKT`, `SimpleKT`, `QIKT`, `LBKT` 六个主流 KT 模型。

### 1.2 工业级训练引擎 (`BaseTrainer`)
- **逻辑复用**：将 5 折交叉验证（GroupKFold）、早停机制（Early Stopping）、AMP 自动混合精度等复杂逻辑封装在 `BaseTrainer` 中。
- **评估标准化**：所有模型强制使用 `BCEWithLogitsLoss` 并统一计算 `AUC`, `ACC`, `RMSE` 等指标，确保实验对比的绝对公平。

### 1.3 分布式分布式调参系统 (Optuna + SQLite)
- **贝叶斯优化**：集成 Optuna 代替传统网格搜索，显著提升寻找最优超参的效率。
- **多机/多进程协同**：支持通过 SQLite 数据库实现多任务同步。开启 `optuna-dashboard` 可实时监控实验动态。
- **YAML 驱动**：通过独立配置文件管理每个模型的搜索空间（Search Space），调参过程无需修改代码。

## 3. 核心工程优化 (Engineering Highlights)

### 1. 完善的掩码隔离 (Mask Isolation)
在 Transformer 类模型（AKT, SimpleKT）中，强制物理隔离了**因果掩码 (Causal Mask)** 与 **填充掩码 (Padding Mask)**。避免了模型在训练中由于 Padding 权重瓜分注意力导致的梯度噪声。

### 2. 回归 Logit 预测
所有模型 Forward 函数均输出原始权值 (Logits)，由 `BCEWithLogitsLoss` 结合 `LogSumExp` 技巧处理。相比 Sigmoid 转二分类，这种方式极大缓解了深度模型在早期的梯度消失和数值溢出问题。

### 3. 多分支自动化管理
- **Study Naming**: 自动生成符合规范的训练组名 `[model]_[dataset]_e[epoch]_k[fold]`。
- **Best Params Export**: 调参结束后自动将最优参数导出为 JSON，可直接供主训练流程加载，实现了“调优-产出结果”的闭环。
