# 数据处理流程重构总结

## 重构目标

将 `data_process.py` 从硬编码的数据集特定逻辑，转变为**纯配置驱动**的通用数据处理管道。

## 核心改进

### 1. **配置驱动的列名标准化**

**问题：** 原先每个数据集都有不同的列名（`user_id`, `problem_id`, `correct` vs `question_id`, `label`），导致代码中充斥着 `if dataset == 'xxx'` 的条件判断。

**解决方案：** 
- 在 TOML 配置文件中定义 `column_map`，将原始列名映射为框架标准列名
- 数据加载后**立即重命名**，后续所有逻辑只使用标准列名

**标准列名定义：**
```toml
[column_map]
uid = "user_id"           # 用户 ID
qid = "problem_id"        # 题目 ID  
sid = "skill_id"          # 技能 ID
label = "correct"         # 答题正确性
timestamp = "order_id"    # 时间戳
rt = "ms_first_response"  # 答题时长
```

### 2. **动态过滤器（集成进 Adapter）**

**问题：** 数据集特定的过滤逻辑（如 ASSIST09 需要 `original==1`）硬编码在代码中。

**解决方案：**
```toml
[filters]
original = 1  # 仅保留 original=1 的记录
```

过滤逻辑现在由 `KTDataAdapter.load_from_toml()` 内部完成，会读取 TOML 中的 `filters` 字段并调用内部 `_apply_filters()`：
```python
filters = toml_config.get("filters", {})
df = KTDataAdapter._apply_filters(df, filters)
```

### 3. **通用数据清洗逻辑**

**修改前：**
```python
if 'correct' not in df.columns:
    raise ValueError("...")
user_counts = df['user_id'].value_counts()
```

**修改后：**
```python
if label_col not in df.columns:
    raise ValueError("...")
user_counts = df[user_col].value_counts()
```

所有函数都支持参数化的列名（默认为标准列名）。

## 使用示例

### 处理 ASSIST09

```bash
python data_process.py --dataset assist09 --ratio 1.0 --stride 0
```

**流程：**
1. 加载 `config/datasets/assist09.toml`
2. 读取 CSV 并将 `user_id → uid`, `problem_id → qid`, `correct → label`
3. 应用过滤器：`df = df[df['original'] == 1]`
4. 执行通用清洗、序列构建、Parquet 保存

### 处理 EdNet

```bash
python data_process.py --dataset ednet_kt1 --ratio 1.0
```

**流程：**
1. 加载 `config/datasets/ednet_kt1.toml`
2. 读取 CSV 并将 `question_id → qid`, `elapsed_time → rt`
3. 无需过滤器（EdNet 已通过 `ednet_sample.py` 预处理）
4. 执行相同的通用逻辑

## 架构优势

### ✅ 消除条件判断

**重构前：**
```python
if src_config.dataset.DATA_NAME == 'assist09':
    if 'original' in df.columns:
        df = df[df['original'] == 1]
elif src_config.dataset.DATA_NAME == 'ednet':
    # 其他逻辑
```

**重构后：**
```python
# 完全不需要 if...else...
filters = dataset_config.get('filters', {})
df = apply_filters(df, filters)
```

### ✅ 新增数据集成本极低

支持新数据集只需：
1. 创建 `config/datasets/new_dataset.toml`
2. 定义 `column_map` 和可选的 `filters`
3. 无需修改任何代码

### ✅ 代码可读性提升

所有业务逻辑使用统一的语义化列名（`uid`, `qid`, `label`），无需猜测 `user_id` vs `student_id` 的差异。

## EdNet 特殊处理

EdNet 数据集由于原始数据质量问题（13万用户，分布极度不均），需要独立的预处理脚本：

```bash
# Step 1: 预处理（合并、采样、计算正确率）
python data_utils/ednet_sample.py

# Step 2: 标准处理流程
python data_process.py --dataset ednet_kt1
```

**职责分离：**
- `ednet_sample.py`：数据清洗 & 采样（"碎石" → "砖块"）
- `data_process.py`：序列构建 & 特征工程（"砖块" → "模型可食用数据"）
