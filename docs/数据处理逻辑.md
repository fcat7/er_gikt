数据处理逻辑

```
Step 1: 数据加载与标准化 → Step 2: 数据集特定过滤 → Step 3: 过滤短序列
    ↓
Step 4: 处理 Skill ID 分隔符 → Step 5: 分层采样 (ratio < 1.0，长度+正确率) 
    ↓
Step 6: 用户级 Train/Test 分割 (8:2, 严格无交集)
    ↓
Step 7: 全景图结构构建 (DataBuilder)
    ├─ ID 映射: problem/skill/user (存为独立 JSON)
    ├─ 邻接矩阵: 构建并**稀疏化保存**为 qs_table, qq_table, ss_table.npz
    └─ Domain Map: 基于 Ward 层次聚类或手动配置生成技能领域映射
    ↓
Step 8: 序列构建 (Parquet)
    ├─ 格式: 转换为 List of Dicts，使用 PyArrow 引擎保存为 parquet
    ├─ 时间特征提取 (Time Engineering):
    │    ├─ t_interval: 距该题涉及 Skill 上次练习的时间间隔，经 log(1+Δt) + tanh 归一化
    │    └─ t_response: 学生作答时间 vs 该题全量中位时间的比率，经 tanh 归一化且在 [0,1] 截断
    ├─ eval_mask 生成:
    │    ├─ Train: 若 stride > 0 使用滑动窗口，eval_mask 标记非重合部分
    │    └─ Test: 长序列强制 50% 重叠切分，生成 eval_mask 解决冷启动
    ↓
Step 9: 题目统计特征计算
    └─ 计算 Difficulty, Discrimination, RT，归一化后存为 q_features.npy
    ↓
Step 10: 统一元数据生成 (metadata.json)
    └─ 汇总 n_user/q/s、各文件索引、字段定义，实现一键式数据加载封装
```

训练逻辑：

```text
初始化
├─ 加载模型 (GIKT)
├─ 加载数据 (Dataset with eval_mask)
├─ K-Fold 划分
│  ├─ 若 groups 存在 → GroupKFold (防窗口化数据泄露)
│  └─ 否则 → 标准 KFold

For each Fold:
├─ 重新初始化模型 & 优化器
├─ For each Epoch:
│  ├─ Training:
│  │  ├─ 前向 + 错位预测 (y_hat[:, 1:])
│  │  ├─ final_mask = mask_valid & eval_mask_valid
│  │  ├─ Loss/Acc/AUC 仅在 final_mask 上计算
│  │  └─ 收集全量 targets/probs 计算 Global AUC
│  │
│  ├─ Validation:
│  │  ├─ 同 Training 逻辑 (eval_mask 过滤)
│  │  └─ Global AUC 计算
│  │
│  └─ Early Stopping (per fold)
│
├─ Fold 完成后再评一次测试集 (用 final_mask)
└─ 记录 Fold AUC

最后: 输出 CV 平均 AUC ± Std Dev
```
